Confirmation and Exploratory Testing

Introduction

In this exercise, you’ll be working with the Employee Manager app and the Home Loan Wizard app doing confirmation and exploratory testing.

Instructions

Employee Manager

First, let’s do some confirmation testing with Employee Manager.

  • Take each bug you’ve found in your testing for Employee Manager so far and turn that bug into a confirmation test.

  • You should have at least 1 test per bug to ensure your coverage is sufficient.

  • Be sure to link each confirmation test to the original bug! Traceability is still important!

Home Loan Wizard

Next up we’ll be looking at a new site for some Exploratory Testing!

  • Clone the following repository: Home Loan Wizard.

  • In your terminal, in the home-loan-wizard directory, run: npm i.

  • Then run npm start.

  • Review the requirements here.

  • Start with some exploratory testing, then create a test plan. Also, specifically regarding the Distribution Rules, create a decision table tracking the various inputs/conditions and their resulting outputs/actions.

When you’re done

Make sure to add all of your materials to a GitHub repo.
